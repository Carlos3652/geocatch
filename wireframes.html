GEOCATCH — UX/UI AUDIT REPORT

Character Select — Issues

  • No game title or branding shown on screen
  • No visible selection state on default trainer
  • Trainers float on plain black — no world context
  • Keyboard-only selection, no mouse click support
  • Instructions are one cramped unreadable string
  • How to Play and High Scores not visually separated

HUD During Play — Issues

  • Score/timer have no backing panel → unreadable on grass
  • Timer never signals urgency when time runs low
  • No feedback when catching a creature (silent disappear)
  • Bomb hit gives no visual feedback (silent −100 pts)
  • No proximity label showing creature name/points
  • Inventory is built in code but never shown on screen

Game Over Screen — Issues

  • No "NEW HIGH SCORE" indicator
  • Name entry says "saved!" before Enter is pressed (bug)
  • No R-to-skip path shown before name entry completes
  • No caught creature recap shown
  • Plain black background — missed cinematic opportunity

Accessibility — Failures

  • White text on #228B22 grass fails contrast near trees
  • Bomb penalty has no shape/icon redundancy (CVD risk)
  • Timer urgency is color-only (red/green colorblind issue)
  • No mouse interaction anywhere in the game
  • Segoe UI Emoji font is Windows-only (cross-platform bug)

Streamlit Version — Assessment

  • Debug "app working" banner visible to end users (remove)
  • No real game loop — Streamlit rerenders on every click
  • Map was removed, core fantasy of game is absent
  • No character select, no encounter screen, no timer
  • Architecture is wrong for real-time gameplay
  • Recommend: full rethink or focus solely on Pygame version

World Design — Issues

  • Trees randomized each session — no mental map possible
  • Creatures can spawn inside the lake (immersion break)
  • Bombs/rocks use emoji — may not render cross-platform
  • Road color (brown) reads as dirt, not Kansas asphalt
  • No visual indicator of catch radius (55px invisible circle)
