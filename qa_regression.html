HIGH-02 HIGH [GameOver] Firework particles do not fade -- computed _fw_alpha is calculated but never applied
Description

Lines 1077-1082 compute a per-particle fade value based on remaining lifetime and then draw the particle. However, the alpha value is never applied to the drawn circle: 

# Line 1078 -- alpha computed correctly
_fw_alpha = max(0, min(255, int(255 * fw["life"] / fw["max_life"])))

# Line 1079 -- (*fw["color"],) unpacks a plain RGB 3-tuple; the trailing comma adds nothing.
# pygame.draw.circle on a non-SRCALPHA surface ignores any alpha in the color tuple.
pygame.draw.circle(screen, (*fw["color"], ), (int(fw["x"]), int(fw["y"])), fw["r"])

# Line 1080-1082 -- _fw_alpha is used only as a threshold condition, never as a color component
if _fw_alpha < 200:
    pygame.draw.circle(screen, (*fw["color"],),
        (int(fw["x"] - fw["vx"]*0.02), int(fw["y"] - fw["vy"]*0.02)), max(1, fw["r"]-1))

Because screen is a plain (non-SRCALPHA) surface created via pygame.display.set_mode(), pygame.draw.circle ignores any fourth alpha component in the color argument. The result is that every firework particle maintains full brightness from spawn until its life reaches zero, at which point it vanishes with no fade-out whatsoever. The intended "particle decay" visual effect is completely absent. _fw_alpha is a dead variable.

Steps to Reproduce

1. Achieve a new high score on a fresh run (score > existing minimum, or any score if leaderboard has fewer than 5 entries).
2. Observe firework particles on the game-over screen.
3. All particles maintain full color intensity and abruptly disappear when their life timer expires. No fade-out is visible.

Expected
Firework particles visibly fade from full brightness to black (background color) over their lifetime, producing a natural decay effect that smoothly removes particles from the scene.
Actual
Particles render at full opacity for their entire lifetime and disappear abruptly on the frame their life value drops to zero or below. _fw_alpha is computed but discarded.
Recommended Fix

Since screen is a non-SRCALPHA surface, simulate fade by linearly interpolating each color channel toward the background color (14, 16, 32): 

for fw in _go_fireworks:
    _t = fw["life"] / fw["max_life"]   # 1.0 = freshly spawned, 0.0 = dying
    _bg = (14, 16, 32)
    _fc = tuple(int(_bg[i] + (fw["color"][i] - _bg[i]) * _t) for i in range(3))
    pygame.draw.circle(screen, _fc, (int(fw["x"]), int(fw["y"])), fw["r"])
    # Trail dot (shown when particle is more than 20% faded)
    if _t < 0.8:
        pygame.draw.circle(screen, _fc,
            (int(fw["x"] - fw["vx"]*0.02), int(fw["y"] - fw["vy"]*0.02)),
            max(1, fw["r"] - 1))

Expected

Actual

Description
