CRIT-01 — save_high_score() crashes on OSError, losing player data

geocatch_pygame.py : 163–170 | Feature #11 / score persistence

save_high_score() opens highscores.txt for write with no try/except guard. Any OSError (read-only filesystem, disk full, permission denied, antivirus lock on Windows) raises an unhandled exception that terminates the game loop while the player is on the game-over screen. The score is already appended to high_scores in memory before the crash, so the in-memory state is half-updated. On the next launch the file is unchanged, silently discarding the new score. This path is reachable on any machine where the working directory is not writable — common when the game is run from Program Files or a network share.

Bug — lines 163–170
def save_high_score(name, sc):
    global high_scores
    high_scores.append((name, sc))
    high_scores.sort(key=lambda x: x[1], reverse=True)
    high_scores = high_scores[:5]
    with open("highscores.txt", "w") as f:   # OSError crashes game here
        for n, s in high_scores:
            f.write(f"{n} {s}\n")
Fix
def save_high_score(name, sc):
    global high_scores
    high_scores.append((name, sc))
    high_scores.sort(key=lambda x: x[1], reverse=True)
    high_scores = high_scores[:5]
    try:
        with open("highscores.txt", "w") as f:
            for n, s in high_scores:
                f.write(f"{n} {s}\n")
    except OSError:
        pass  # Scores remain in memory for this session; silently skipped on next launch

HIGH-01 — set_alpha() is silently ignored on SRCALPHA surfaces — fade-in never renders

geocatch_pygame.py : 1089–1094 | Features #9 Fade-in, #5 Fader behavior

The fade-in and fader-behavior alpha effects are both routed through _fade_sticker_surf.set_alpha(_c_alpha). However, _fade_sticker_surf was created with pygame.SRCALPHA (line 280). Per-pixel alpha surfaces and surface-wide alpha are mutually exclusive in Pygame: the documentation explicitly states "the per-pixel alpha flag and the surface alpha are mutually exclusive" and set_alpha() has no effect on SRCALPHA surfaces. The call at line 1092 silently does nothing. All creatures appear at full opacity during their entire fade-in window, and the Shadow Phantom fader effect does not dim at any distance. The reset call at line 1094 (set_alpha(255)) is likewise a no-op.

The correct approach is to blit the sticker onto a non-SRCALPHA surface set to the desired surface-wide alpha, or to pre-multiply the alpha into each pixel manually. The simplest fix is to use a regular (non-SRCALPHA) intermediate surface and call set_alpha() on it, since the sticker source already carries per-pixel alpha that will be composited during the blit.

Bug — lines 280, 1090–1094

# At module level, replace line 280:

MED-01 — Score counter animation skips non-multiples of 10 — broken for streaks

geocatch_pygame.py : 825 | Features #3, #6 / game-over animation

The score roll-up animation advances by computing int(_go_anim_timer / 0.05) * 10. This means the counter always jumps in steps of exactly 10 and never displays values between multiples of 10. For a score of 67, the counter displays …60 → 67, which looks like a glitch compared to the smooth animation leading up.

LOW-02 — Mono sound buffer may mismatch Pygame mixer channel count on some platforms

geocatch_pygame.py : 9, 43–54 | Feature #1 Sound

pygame.mixer.pre_init(44100, -16, 1, 512) requests a mono mixer. The tone generator correctly creates a mono array.array('h') buffer and passes it directly to pygame.mixer.Sound(buffer=buf). This works correctly on Windows and most Linux setups. However, on some macOS and Linux configurations the system audio driver silently upgrades the channel count to 2 even when 1 is requested — causing Pygame to report channels=2 after pygame.init(). In that scenario, a mono buffer supplied to Sound(buffer=) is interpreted as half-length stereo, producing a sound at half the intended duration with garbage in the second channel. Wrapping in the existing try/except makes this survivable, but the audio will be subtly wrong rather than silently absent.

Defensive fix — verify channel count post-init and duplicate if needed
def _gen_tone(freq, duration_ms=100, volume=0.3, freq_end=None):
    sr = 44100
    n = int(sr * duration_ms / 1000)
    if n == 0:
        return None
    channels = pygame.mixer.get_init()[2]  # actual channel count after init
    buf = _array.array('h', [0] * n * channels)
    fe = freq_end if freq_end else freq
    for i in range(n):
        t = i / sr
        f = freq + (fe - freq) * (i / n)
        fade = 1.0
        fs = int(n * 0.75)
        if i > fs and n > fs:
            fade = (n - i) / (n - fs)
        val = int(volume * 32767 * math.sin(2 * math.pi * f * t) * fade)
        val = max(-32768, min(32767, val))
        for ch in range(channels):
            buf[i * channels + ch] = val
    return pygame.mixer.Sound(buffer=buf)
